# AR Images

AR Images is a browser-based WebAR project that lets users upload image and video pairs, compile those images into a MindAR target library, and view anchored video overlays on top of the detected real-world image through the device camera.

## Stack

- Frontend: HTML, CSS, vanilla JavaScript
- WebAR: A-Frame + MindAR image tracking
- Backend: Node.js + Express + Multer
- Storage: Local filesystem for prototype uploads and compiled `.mind` target bundle
- Database: JSON file persistence in `data/mappings.json`

## Implementation Steps

1. The dashboard uploads an image and an MP4 video to the Express API.
2. The backend stores file URLs plus metadata for each mapping in `data/mappings.json`.
3. After mappings are created or updated, the dashboard uses `window.MINDAR.Compiler()` in the browser to compile all target images into one combined `.mind` file.
4. The compiled `.mind` file is posted back to the backend and saved as `public/uploads/targets/current.mind`.
5. The AR viewer loads that target bundle, opens the camera with MindAR, and renders each matching video through A-Frame when its image target is found.

## API Overview

- `GET /api/health` - basic health check
- `GET /api/mappings` - fetch all mappings and current target bundle info
- `POST /api/mappings` - create a new mapping with `image` and `video`
- `PUT /api/mappings/:id` - update name, image, or video
- `DELETE /api/mappings/:id` - remove a mapping
- `POST /api/targets` - save compiled `.mind` target bundle generated by the dashboard

## Local Run

1. Install dependencies:

```bash
npm install
```

2. Start the server:

```bash
npm start
```

3. Open the dashboard:

```text
http://localhost:3000
```

4. Upload one or more mappings, then let the dashboard rebuild the target file.
5. Open `http://localhost:3000/viewer.html` on a mobile device on the same network or directly on the device.

## Mobile Testing Notes

- Use HTTPS in production. iOS Safari and Android Chrome need secure context for camera access, except on `localhost`.
- Target images should contain rich features and contrast. Plain logos or low-detail artwork are harder to track reliably.
- Videos are muted by default to satisfy mobile autoplay rules. If you need sound, add a tap-to-unmute pattern.

## Deploy

### Vercel

This project can deploy to Vercel as a Node.js app, but keep in mind that local filesystem uploads are ephemeral on serverless platforms. For a production deployment:

1. Move uploaded files and compiled target bundles to cloud storage such as Firebase Storage, Cloudinary, or S3.
2. Replace JSON persistence with a hosted database such as Firestore, Supabase, or PostgreSQL.
3. Deploy the Express server as a Vercel serverless function or a separate backend service.

### Netlify

Netlify is best used for the static frontend here. If you want Netlify:

1. Host `public/` on Netlify.
2. Move the Express API to Netlify Functions or another backend host such as Render, Railway, or Firebase Functions.
3. Store uploads and `.mind` files in cloud storage instead of the local filesystem.

## Sample Demo Setup

For a quick demo:

1. Use a high-detail poster, flyer, magazine cover, or product package image.
2. Upload a short muted MP4 clip that matches that target.
3. Rebuild the target file.
4. Open the AR viewer on your phone and scan the printed or on-screen target image.

## Known Production Follow-Ups

- Move uploads to cloud storage for durable deploys
- Add authentication if multiple users need isolated libraries
- Add drag-and-drop uploads and richer progress feedback
- Add explicit video sizing controls per target for more art-directed overlays
